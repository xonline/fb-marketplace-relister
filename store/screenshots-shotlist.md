## Shot 1 — The On-Page Button (First Impression)

**Feature shown:** Injected Relist button on the Selling page

**What's on screen:**
- Facebook Marketplace `/marketplace/you/selling` page in Chrome
- A real-looking listing card (e.g. a sofa or PlayStation — generic consumer goods, not personal items)
- A clearly visible "Relist" button injected by the extension, positioned next to the listing's existing Edit / Mark as Sold buttons
- Optionally: a subtle badge on the listing showing "Listed 4 days ago" to set up the "why relist" context
- Browser toolbar visible with the extension icon (shows the extension is active)

**Caption/overlay text (white text on semi-dark overlay bar at bottom):**
> "Relist button appears right on your Marketplace Selling page — no extra steps"

**Design note:** The button should look native to Facebook's UI — same rounded style as the existing Edit button, but a distinct color (teal or green to differentiate). Highlight it with a subtle glow or callout arrow so the eye goes directly to it.

**Competitive angle:** No other extension injects the button inline on the Selling page — Relistify and AutoListing require opening a popup first. This shot shows we go one step further.

## Screenshot Production Notes

- **Resolution:** 1280×800 px minimum. PNG preferred.
- **Facebook UI:** Use a real or near-real mockup of the Facebook Marketplace interface. Avoid using actual user personal data, profile photos, or real listing prices. Use placeholder names (e.g. "John S.") and generic product photos.
- **Extension UI:** Take from a real extension popup if the UI is built; otherwise design mockups at pixel-perfect dimensions.
- **Fonts in overlays:** Use a clean sans-serif (Inter, DM Sans, or similar). Not Facebook's proprietary font.
- **Dark mode:** Consider dark-mode popup screenshots for shots 2 and 3 — they read with higher contrast as store images.
- **Device frame:** Chrome Web Store does not add a device frame — you should add your own browser chrome (Chrome window top bar) for context, or go full-bleed with the UI only.

## Ordering on Store

Chrome Web Store shows screenshots in a horizontal scroll. First screenshot = hero impression. Recommended ordering:

1. Shot 1 (on-page button) — instant clarity on what the extension does
2. Shot 3 (bulk relist) — shows power
3. Shot 4 (schedule + price drop) — shows free premium features
4. Shot 2 (real-time progress) — builds trust
5. Shot 5 (before/after) — closes the sale with emotional outcome
